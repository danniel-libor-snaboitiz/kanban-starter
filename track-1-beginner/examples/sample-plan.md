# Implementation Plan — Kanban with Comments & @Mentions

_Generated by planning subagent, then edited for clarity._

## Assumptions

- Starting from `starter-repos/laravel-starter/` on branch `main`.
- User model already exists; we add Board / Column / Card / Comment / Notification.
- No websockets — page-refresh model is fine for the beginner track.

## Steps

### 1. Data layer — Boards, Columns, Cards (60 min)

- Migrations: `boards (user_id, name)`, `columns (board_id, name, position)`, `cards (column_id, title, description, position, assignee_id)`.
- Models: relationships and fillable arrays.
- Factories for testing.
- **Depends on:** nothing.
- **Risk:** none.

### 2. Board CRUD + auth policy (30 min)

- Routes + `BoardController`.
- `BoardPolicy`: only owner can view/edit/delete.
- Failing feature test first: `test_user_cannot_access_others_board`.
- **Depends on:** 1.

### 3. Column CRUD within a board (25 min)

- Nested routes: `/boards/{board}/columns`.
- Reorder endpoint: `PATCH /columns/reorder` with an ordered array of ids.
- Failing tests first.
- **Depends on:** 2.

### 4. Card CRUD (30 min)

- Under `/columns/{column}/cards`.
- Drag-drop endpoint: `PATCH /cards/{card}/move` with `{ column_id, position }`.
- Failing tests first.
- **Depends on:** 3.

### 5. Mention parser service (20 min)

- `App\Services\MentionParser::parse(string $body): array` returns an array of `['@username', User|null]`.
- Regex: `/@([a-zA-Z0-9_.]+)/`.
- **TDD required.** Test cases: no mentions, one mention, two mentions, dot username, unknown username returns `null`.
- **Risk:** low; pure logic.
- **Depends on:** 1 (User model).

### 6. Comment model + posting (25 min)

- Migration: `comments (card_id, user_id, body, created_at)`.
- `CommentController@store` parses mentions after save.
- Failing tests first: comment creation + mention parsing hook.
- **Depends on:** 4, 5.

### 7. Notifications (25 min)

- Migration: `notifications (user_id, type, data JSON, read_at, created_at)` with index on `(user_id, read_at)`.
- Notification created on mention.
- `/notifications` route + view; `PATCH /notifications/{id}/read`.
- Failing tests first.
- **Depends on:** 6.

### 8. UI — Blade + Alpine drag-drop (60 min)

- Board view with columns and cards.
- Drag-drop via native HTML5 drag events + Alpine.
- Comment modal on card click.
- Nav-bar unread notifications badge.
- **Depends on:** 2-7.
- **Risk:** medium; drag-drop always takes longer than you'd think.

### 9. HTML workflow report + polish (20 min)

- Fill in report template from `examples/sample-html-report.html`.
- Take screenshots.
- Push to GitHub.

## Estimated total

~4h 15min. Cut scope on step 3 (skip reorder) if you're over budget.
